## 更新内容

### ✅ 改进 1：NVD API Key 使用环境变量

#### 改动说明
- **移除了** GUI 中的 API Key 输入框
- **改为** 自动使用系统环境变量 `NVD_API_KEY`
- **新增** API Key 状态显示（绿色✓或橙色⚠️）

#### 优势
1. **更安全**：API Key 不在界面中明文显示
2. **更方便**：一次配置，长期使用
3. **更快捷**：无需每次手动输入

#### 界面变化
**之前：**
```
[采集范围] [7天▼] | NVD API Key: [****输入框****] | [▶ 采集]
```

**现在：**
```
[采集范围] [7天▼] | ✓ API Key 已配置 | [▶ 采集]
或
[采集范围] [7天▼] | ⚠ 未配置 API Key（速度较慢） | [▶ 采集]
```

#### 如何设置环境变量

详见文档：`如何设置API_Key.md`

**Windows PowerShell（推荐）：**
```powershell
# 临时设置（当前会话）
$env:NVD_API_KEY="your-api-key-here"

# 永久设置：系统设置 → 环境变量
```

**Linux/macOS：**
```bash
# 临时设置
export NVD_API_KEY="your-api-key-here"

# 永久设置：添加到 ~/.bashrc
echo 'export NVD_API_KEY="your-api-key-here"' >> ~/.bashrc
source ~/.bashrc
```

#### 日志提示
```
✓ 使用环境变量 API Key，采集速度更快
或
⚠ 未配置 API Key，采集速度较慢
提示：设置环境变量 NVD_API_KEY 可提升速度 10 倍
```

### 首次使用流程

#### 1. 设置 NVD API Key（可选但推荐）

```bash
# Windows PowerShell
$env:NVD_API_KEY="your-api-key-here"

# Linux/macOS
export NVD_API_KEY="your-api-key-here"
```

#### 2. 启动程序

```bash
# 激活虚拟环境
source /D/AI/cursor/starone/.venv/Scripts/activate

# 启动程序
python cve_integrated_gui.py
```

#### 3. 生成 Dell 示例数据

1. 切换到 "🏢 Dell 安全公告" 标签页
2. 点击 "▶ 生成示例 Dell 数据"
3. 等待生成完成（几秒钟）

#### 4. 采集 NVD CVE 数据（可选）

1. 切换到 "📊 NVD CVE 数据" 标签页
2. 选择天数范围（建议从 7 天开始）
3. 点击 "▶ 采集 NVD 数据"

#### 5. 查看 CVE-Dell 关联

1. 切换到 "🔗 CVE-Dell 关联" 标签页
2. 点击 "🔄 刷新关联数据"
3. 双击查看详细信息

## 常见问题

### Q1: 为什么移除了 API Key 输入框？
A: 为了安全性和便利性。环境变量方式更安全，且一次配置即可长期使用。

### Q2: 如何知道 API Key 是否配置成功？
A: 界面上会显示：
- ✓ API Key 已配置（绿色）= 成功
- ⚠ 未配置 API Key（橙色）= 未设置

### Q3: Dell 数据是真实的吗？
A: 示例数据采用真实的 Dell 安全公告格式，包含真实的产品型号和解决方案结构，但 CVE 编号是示例编号。

### Q4: 可以添加真实的 Dell 数据吗？
A: 可以！手动创建 JSON 文件（参考示例格式），或访问 Dell 官网手动获取数据。详见 `Dell数据说明.md`。

### Q5: 升级后需要重新安装吗？
A: 不需要。直接使用新版本的 `cve_integrated_gui.py` 即可。
