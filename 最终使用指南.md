## 🚀 快速开始

```bash

start_cve_gui.bat
```

**方式 B：手动启动**
```bash
source /D/AI/cursor/starone/.venv/Scripts/activate
python cve_integrated_gui.py
```

### 2. 基本使用流程

#### 步骤 1：采集 NVD CVE 数据（可选）

1. 切换到 "📊 NVD CVE 数据" 标签页
2. 选择天数范围（建议从 7 天开始测试）
3. （可选）输入 NVD API Key
   - 申请地址：https://nvd.nist.gov/developers/request-an-api-key
   - 免费，速度提升 10 倍
4. 点击 "▶ 采集 NVD 数据"
5. 等待采集完成

**注意：** 如果遇到网络问题，可以直接使用步骤 3 加载本地数据。

#### 步骤 2：采集 Dell 安全公告（演示）

1. 切换到 "🏢 Dell 安全公告" 标签页
2. 点击 "▶ 采集 Dell 安全公告"
3. 系统会自动使用示例数据（包含 5 条 Dell 安全公告）
4. 数据会自动保存到 `cve_data/dell_advisories_*.json`

**说明：** 由于 Dell RSS 已停用，系统使用高质量的示例数据，格式与真实 Dell 公告完全一致。

#### 步骤 3：查看 CVE-Dell 关联

1. 确保已有 NVD 和 Dell 数据（可以使用加载本地数据）
2. 切换到 "🔗 CVE-Dell 关联" 标签页
3. 点击 "🔄 刷新关联数据"
4. 双击任一条目查看详细信息

**查看内容包括：**
- CVE 基本信息（编号、严重等级、CVSS 评分、描述）
- Dell 公告信息（公告 ID、标题、发布日期）
- **Dell 受影响产品型号**（如 PowerEdge R750、OptiPlex 7090）
- **Dell 官方解决方案和操作方法**
- 参考链接（NVD 链接、Dell 官方链接）

### 3. 离线使用（无需网络）

程序启动时会自动加载最新的本地数据文件，也可以手动加载：

1. 点击 "📁 加载本地数据" 按钮
2. 选择数据文件：
   - NVD CVE 数据：`cve_data/nvd_cves_*.json`
   - Dell 公告数据：`cve_data/dell_advisories_*.json`
3. 数据立即显示在相应页面
