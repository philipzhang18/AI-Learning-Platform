#### 3.2 CWE 标签覆盖不足（严重度：中）
**问题描述**:
`dsa_prediction_series.py` 的 `CWE_MAPPING` 字典包含 35 个 CWE 类型，但实际测试发现：
- PowerScale 预测返回 CWE-606、CWE-385 等标记为"其他/未分类"
- 知识图谱包含 53 个 CWE 节点，部分未在映射表中

**示例缺失 CWE**:
- CWE-606（untrusted search path）
- CWE-385（covert timing channel）
- CWE-776（XML entity expansion）
- CWE-250（execution with unnecessary privileges）
- CWE-35（path traversal）

**影响**:
用户界面显示"其他/未分类"，降低预测可解释性

**建议**:
扩展 `CWE_MAPPING` 至 MITRE Top-100 或从 `cves.data.weaknesses` 动态构建标签映射

#### 3.3 低置信度标记（严重度：低）
**问题描述**:
PowerScale OneFS 预测未触发 `low_confidence` 警告（279 条历史 DSA），但 iDRAC 9 同样未触发（历史样本数未显示）

**验证点**:
当历史 DSA < 3 条时应设置 `low_confidence = True`，代码逻辑正确：
```python
low_confidence = historical_count < self._MIN_CONFIDENCE_SAMPLES  # 3
```

**测试覆盖建议**:
添加边界测试用例（如冷门产品系列，历史 DSA 0-2 条）验证低置信度标记

#### 3.4 版本匹配逻辑 Bug（严重度：中，已修复）
**问题描述**:
在测试中发现"PowerScale OneFS"系列查询返回 279 条 DSA，但早期版本（未打补丁）返回 0 条。

**根本原因**:
`_query_series_history()` 原逻辑：
```python
if any(series_lower in line.lower() for line in classified):
    is_match = True
```
当 `series_lower = "powerscale onefs"`，无法匹配 `classify_dsa()` 返回的 `"PowerScale / Isilon (NAS)"`

**修复方案**（已集成）:
代码第 340-356 行已改为三级匹配：
1. 标题原文包含系列名（`"powerscale onefs" in title_lower`）
2. 系列首词匹配分类标签（`"powerscale" in "powerscale / isilon (nas)"`）
3. 标签首词匹配系列名（反向匹配）

**验证结果**: ✅ PowerScale OneFS 现匹配 279 条 DSA

#### 7.3 集成示例代码
```python
from risk.dsa_prediction_series import ProductSeriesPredictor
from datetime import datetime

# 初始化预测器
predictor = ProductSeriesPredictor(
    db_path="cve_data/cve_database.db",
    eoss_oe_path="risk/eoss_data.json",
    now=datetime.now()
)

# 预测
forecast = predictor.forecast_series(
    product_series="PowerScale OneFS",
    version="9.8.0.0",
    forecast_days=90
)

# 展示结果
print(f"风险等级：{forecast.risk_level}")
print(f"概率：{forecast.probability:.1%}")
for cwe_id, label, pct in forecast.likely_cwes:
    print(f"  {cwe_id} {label} {pct:.1f}%")
```
