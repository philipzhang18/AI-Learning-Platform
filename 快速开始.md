# CVE 漏洞监控系统 - 快速开始

## 🚀 5 分钟快速开始

### 步骤 1：设置 NVD API Key（可选但推荐）

**Windows PowerShell:**
```powershell
$env:NVD_API_KEY="your-api-key-here"
```

**Linux/macOS:**
```bash
export NVD_API_KEY="your-api-key-here"
```

> 💡 **提示：** 没有 API Key？访问 https://nvd.nist.gov/developers/request-an-api-key 免费申请（速度提升 10 倍！）

### 步骤 2：启动程序

```bash
# 激活虚拟环境
source /D/AI/cursor/starone/.venv/Scripts/activate

# 启动程序
python cve_integrated_gui.py
```

或使用启动脚本：
```bash
# Linux/macOS
./start_cve_gui.sh

# Windows
start_cve_gui.bat
```

### 步骤 3：生成 Dell 示例数据（1 分钟）

1. 点击 **"🏢 Dell 安全公告"** 标签页
2. 点击 **"▶ 生成示例 Dell 数据"** 按钮
3. 等待几秒，完成！

✅ 现在你有了 5 条完整的 Dell 安全公告

### 步骤 4：查看 CVE-Dell 关联

1. 点击 **"🔗 CVE-Dell 关联"** 标签页
2. 点击 **"🔄 刷新关联数据"**
3. **双击任一条目** 查看详细信息

你会看到：
- CVE 详细信息
- **Dell 产品型号**（如 PowerEdge R750）
- **Dell 解决方案**（如更新 BIOS 到版本 X.X.X）
- 详细操作步骤

### 步骤 5：采集真实 NVD 数据（可选）

1. 点击 **"📊 NVD CVE 数据"** 标签页
2. 选择天数范围（建议从 **7 天** 开始）
3. 点击 **"▶ 采集 NVD 数据"**

> ⚠️ **注意：**
> - 无 API Key：7 天数据约需 2-3 分钟
> - 有 API Key：7 天数据约需 10-20 秒

## 💡 使用技巧

### 1. API Key 状态检查
界面会显示：
- **✓ API Key 已配置**（绿色）= 速度快
- **⚠ 未配置 API Key**（橙色）= 速度慢

### 2. 数据保存位置
所有数据自动保存在 `cve_data/` 目录：
```
cve_data/
├── nvd_cves_20251030_163000.json         # NVD CVE 数据
├── dell_advisories_20251030_163100.json  # Dell 安全公告
```

### 3. 离线使用
- 程序启动时自动加载最新数据
- 点击 "📁 加载本地数据" 可选择历史数据
- 完全支持无网络环境使用

### 4. 搜索功能
在各个页面的搜索框输入：
- **CVE ID**：`CVE-2024-1234`
- **严重等级**：`CRITICAL`、`HIGH`
- **产品名称**：`PowerEdge`、`OptiPlex`
